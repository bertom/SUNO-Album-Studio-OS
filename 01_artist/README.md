# Artist folder — your identity lives here

Everything in `01_artist/` defines **who you are as a musician** — not one album, not one genre. Agents read these files before writing lyrics, styles, or SUNO prompts.

## 10-minute setup

1. Open this repo in **Cursor** (or any AI assistant + text editor).
2. Paste the prompt from **`ONBOARDING.md`** at the repo root into a new chat.
3. Run **`/onboard artist`** — answer the interview questions.
4. Skim the filled files below. Edit anything that doesn't sound like you.
5. Next step: **`/album seed`** or drop notes in `02_albums/_album_template/input/` and run **`/orchestrate album`**.

## Files in this folder

| File | Purpose |
|------|---------|
| `artist_identity.md` | Who you are, what you protect, what you avoid |
| `artist_dna.md` | Short DNA paragraph — the body that stays recognizable |
| `style_reference.md` | Full style palette — grows over time |
| `style_reference_summary.md` | Quick lookup for agents (2–4 groove families) |
| `sonic_palette.md` | Instrument and groove pantry |
| `lyric_principles.md` | How you write lyrics |
| `cover_design_guide.md` | Visual principles for album art |
| `feedback_dictionary.md` | Your natural language → creative moves |

## Keep it alive

- After each album: add learnings to `03_global_learnings/`.
- After each winner: add a row to `05_catalog/best_suno_outputs.md` via **`/backfill catalog`** or **`/archive track`**.
- Style families grow — don't force every song into the first family you named.

> Albums may change clothes; the body must remain recognizable.
